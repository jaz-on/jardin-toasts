# Internationalization (i18n)

## Overview

Jardin Toasts is fully internationalized and ready for translation. This document describes the i18n implementation.

## Text Domain

### Domain Name

**Text Domain**: `jardin-toasts`

**Usage**: Consistent throughout entire codebase

**Declaration**: In main plugin file header

---

## Implementation

### Loading Text Domain

**Location**: Main plugin file

**Code**:
```php
add_action('plugins_loaded', 'jb_load_textdomain');

function jb_load_textdomain() {
    load_plugin_textdomain(
        'jardin-toasts',
        false,
        dirname(plugin_basename(__FILE__)) . '/languages/'
    );
}
```

**Hook**: `plugins_loaded` (early enough for translations to load)

## Translation Functions

### Basic Functions

#### __() - Return Translated String

**Usage**: Get translated string (don't echo)

**Example**:
```php
$text = __('Beer Check-ins', 'jardin-toasts');
```

---

#### _e() - Echo Translated String

**Usage**: Echo translated string directly

**Example**:
```php
_e('Import Historical Check-ins', 'jardin-toasts');
```

---

#### esc_html__() - Escape and Return

**Usage**: Get translated and escaped string

**Example**:
```php
$text = esc_html__('Rating System', 'jardin-toasts');
```

---

#### esc_html_e() - Escape and Echo

**Usage**: Echo translated and escaped string

**Example**:
```php
esc_html_e('Settings', 'jardin-toasts');
```

---

#### esc_attr__() - Escape Attribute

**Usage**: Get translated string for HTML attributes

**Example**:
```php
$alt = esc_attr__('Beer photo', 'jardin-toasts');
```

---

### Context Functions

#### _x() - Translated String with Context

**Usage**: Translate string with context to disambiguate

**Example**:
```php
_x('Brewery', 'taxonomy singular name', 'jardin-toasts');
_x('Brewery', 'taxonomy general name', 'jardin-toasts');
```

---

#### _ex() - Echo Translated String with Context

**Usage**: Echo translated string with context

**Example**:
```php
_ex('Style', 'beer style', 'jardin-toasts');
```

---

### Plural Functions

#### _n() - Plural Forms

**Usage**: Handle singular/plural forms

**Example**:
```php
printf(
    _n('%s check-in', '%s check-ins', $count, 'jardin-toasts'),
    number_format_i18n($count)
);
```

---

#### _nx() - Plural with Context

**Usage**: Plural forms with context

**Example**:
```php
_nx('%s item', '%s items', $count, 'check-in count', 'jardin-toasts');
```

## .pot File Generation

### Using WP-CLI

**Command**:
```bash
wp i18n make-pot . languages/jardin-toasts.pot
```

**Options**:
- `--domain=jardin-toasts` - Text domain
- `--exclude=node_modules,vendor` - Exclude directories
- `--headers` - Include headers

---

### Using Poedit

**Manual Process**:
1. Open Poedit
2. Create new translation
3. Extract from sources
4. Save as `.pot` file

---

## Translation Files

### File Structure

```
languages/
├── jardin-toasts.pot              # Template (generated)
├── jardin-toasts-fr_FR.po        # French translation
├── jardin-toasts-fr_FR.mo        # Compiled French
├── jardin-toasts-es_ES.po        # Spanish translation
└── jardin-toasts-es_ES.mo        # Compiled Spanish
```

---

### File Naming

**Format**: `{text-domain}-{locale}.po`

**Examples**:
- `jardin-toasts-fr_FR.po` - French (France)
- `jardin-toasts-fr_CA.po` - French (Canada)
- `jardin-toasts-es_ES.po` - Spanish (Spain)
- `jardin-toasts-de_DE.po` - German (Germany)

---

## Compiling Translations

### .mo File Generation

**Using Poedit**: Automatic on save

**Using msgfmt**:
```bash
msgfmt jardin-toasts-fr_FR.po -o jardin-toasts-fr_FR.mo
```

**Using WP-CLI**:
```bash
wp i18n make-mo languages/
```

## Best Practices

### String Guidelines

- **Complete Sentences**: Translate full sentences, not fragments
- **Context**: Use `_x()` when context is needed
- **Plurals**: Always use `_n()` for countable items
- **Variables**: Use placeholders (`%s`, `%d`) for dynamic content

---

### Code Guidelines

- **Always Include Text Domain**: Every translation function must include `'jardin-toasts'`
- **Escape Output**: Use `esc_html__()` for HTML output
- **No Hardcoded Strings**: All user-facing strings must be translatable

---

### Translation Guidelines

- **Consistency**: Use consistent terminology
- **Context**: Provide context comments for translators
- **Updates**: Update .pot file when adding new strings
